# Apply Migration: Fix validate_segment_configuration RPC Function

**Migration File**: `20251124000100_fix_validate_segment_configuration_rpc.sql`

**Date**: 2025-01-24

---

## Problem

The `validate_segment_configuration` RPC function references the dropped `doctor_segment_configurations` table, causing this error:

```
ERROR: relation "doctor_segment_configurations" does not exist
Code: 42P01
```

This happened because:
1. Migration `20251122140000_update_edge_functions.sql` created the RPC function with `doctor_segment_configurations` table
2. Migration `20251123000000_cleanup_schema_and_add_doctor_templates.sql` dropped that table
3. The RPC function wasn't updated to work with the new template-based architecture

---

## Solution

The migration will:
1. Drop the old RPC function that references the dropped table
2. Create a new RPC function that works with the template-based architecture:
   - Uses `template_segments` table (not `doctor_segment_configurations`)
   - Validates across activated templates (via `doctor_templates` junction table)
   - Checks that required segments are in CORE category

## What Changed

### Old RPC Function (BROKEN)
```sql
-- Referenced dropped table
LEFT JOIN doctor_segment_configurations dsc
    ON dsc.segment_id = sd.id
    AND dsc.doctor_id = p_doctor_id
```

### New RPC Function (FIXED)
```sql
-- Uses template_segments + doctor_templates junction
INNER JOIN template_segments ts
    ON ts.segment_code = sd.segment_code
INNER JOIN doctor_templates dt
    ON dt.template_id = t.id
WHERE dt.doctor_id = p_doctor_id
  AND dt.is_active = TRUE
  AND dt.access_level = 'use'
```
