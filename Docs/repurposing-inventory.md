3. **`doctors.ehr_type_id` is set on counsellor creation** (`doctors.py` via `get_hospital_default_ehr_type_id`). Strip the assignment code before dropping the column.
4. **Billing reads kept lookup tables** (`hospital_medicine_lists`, `*_match_log`). Delete `billing_service.py` but **leave the lookup tables intact**.
5. **`revenue_interventions_service.py` is wired into keep-core `intervention_orchestrator.py`** and reads `hospital_intervention_pricing`. Detach before dropping the pricing table or interventions throw.
6. **`raster_api_service.py` / `raster_lookups.py` / `doctor_lookups.py`** straddle EHR-push (retire) and neonatal keep-repurpose. Confirm neonatal formatters don't call RASTER before deleting.
7. **Alias-view scope**: `CREATE VIEW counsellors AS SELECT * FROM doctors` is updatable only as a 1:1 projection; regenerate after columns drop. **Views rename tables, not FK columns** — `doctor_id`→`counsellor_id` still needs code-level edits (Phase 6).
8. **`audit_service.py` (`phi_audit_log`, HIPAA)** is invoked on access across kept workflows; HIPAA→FERPA + table rename touches every logging read path.
9. **`supabase_service.py` (436KB) is the single chokepoint** — every kept workflow routes table access through it. A missed string silently breaks recording/extraction/merge/Q&A. Highest-risk single edit.
