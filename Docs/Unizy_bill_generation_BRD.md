# BRD: Automated Bill Generation — Unizy Health
**Version:** 1.0  
**Status:** Draft  
**Target:** Multi-specialty hospitals | OP / IP / ER workflows | Cash + TPA/Insurance patients

---

## 1. Problem Statement

- Hospitals manually construct bills from fragmented sources: doctor notes, pharmacy, lab orders, nursing records, and room allocation logs
- Clinical recordings in Unizy already extract ICD codes, procedures, prescriptions, and investigation orders — but this output is not wired to billing
- TPA/insurance billing requires a parallel claim construct mapped to negotiated package rates — currently done manually post-discharge
- Goal: Use Unizy's extraction pipeline to **auto-populate itemized bills** and **generate TPA-ready claim payloads** from the same recording

## 4. Dual-Bill Architecture

Every encounter produces **one itemized bill** (source of truth) and a **mapped TPA claim** (projection layer).

```
RECORDING + STRUCTURED DATA
           │
           ▼
    Unizy EXTRACTION LAYER
    (ICD codes, procedures, investigations,
     prescriptions, consumables, doctor identity)
           │
           ▼
    ITEMIZED BILL ENGINE  ◄──── Master Tables
    (all line items, qty, unit rates)
           │
     ┌─────┴──────┐
     ▼            ▼
 CASH BILL    TPA CLAIM BILL
 (itemized)   (ICD → package mapping
               + TPA tariff master
               + NHCX-ready payload)
```

## 6. Extraction Source Map

```
RECORDING (Audio → Transcript)
        │
        ├── Doctor Identity          → Consultation Fee (fee master lookup)
        ├── Investigations Ordered   → Lab Bill line items
        ├── Imaging Ordered          → Radiology Bill
        ├── Prescription Generated   → Pharmacy Bill (medicine master)
        ├── Procedures Mentioned     → CPT/SNOMED → Procedure Fee
        ├── ICD Codes Generated      → IP Treatment Package / TPA package key
        ├── Consumables Mentioned    → Item Master mapping
        ├── Equipment Ordered        → Equipment Charge Table
        └── Diet/Allied Health       → Respective fee tables

STRUCTURED DATA (Non-transcription)
        ├── Room Allocation Table    → Room Rate × LOS
        ├── Admission/Discharge Timestamps → LOS Calculation
        ├── Doctor Visit Logs        → Daily Round Charges
        ├── Triage Level             → ER Fee Tier
        └── TPA Policy Master        → Package Rates, Sub-limits, Exclusions
```

## 7. TPA / Insurance Billing Logic

### 7.1 Key Principle
- ICD code from transcription = **primary key** into TPA package master
- Itemized bill always generated first; TPA bill is a mapping projection on top

### 7.2 Per-Category TPA Impact

| Category | Cash Logic | TPA Delta |
|---|---|---|
| Consultation | Doctor grade × speciality fee | Often capped per TPA tariff or bundled in package |
| Room | Room category × LOS | Room sub-limit (% of SI/day) → co-pay triggered if over-entitlement |
| Surgery | CPT → procedure fee | ICD → TPA package (negotiated per TPA) — package bundles surgeon + OT + anesthesia + standard drugs |
| Pharmacy | Prescription itemized | Drugs outside formulary or package → patient collection |
| Consumables | SKU × qty | Standard consumables in package; high-value items billed separately |
| Implants | Item + invoice | Always separate + invoice required + PMDA proof |
| Blood Products | Actuals | Billed at actuals with proof |

### 7.3 Pre-Authorization Draft (Unizy Opportunity)
```
Unizy transcription output:
  - ICD code (diagnosis)
  - Procedure name → CPT/ICD-PCS
  - Estimated LOS
  - Room category
        │
        ▼
Auto-draft Pre-Auth form → TPA submission
        │
TPA responds:
  - Approved amount
  - Approved procedure codes
  - Room entitlement
  - Exclusion list
        │
Bill engine locks approved package
+ flags itemized items outside approval
```

## 8. Confidence Scoring — Auto-Bill vs. Flag

| Tier | Confidence | Action | Examples |
|---|---|---|---|
| **Auto-bill** | High | Populate directly, no review | Consultation fee, room rate, registration, triage fee |
| **Auto-suggest** | Medium | Prefill + billing staff reviews | Lab orders, pharmacy from prescription, standard consumables |
| **Flag for manual** | Low / High-value | Do NOT bill until confirmed | Implants, blood products, ICU equipment, any item >₹5,000 |
| **TPA reconcile** | TPA encounter | Diff itemized vs approved package | All TPA claims before submission |

## 9. Master Tables Required

| Master Table | Purpose | Owner |
|---|---|---|
| Doctor Fee Master | Fee by doctor ID + speciality + patient type | Hospital Admin |
| Room Rate Master | Rate by room category + ward | Hospital Admin |
| Procedure / CPT Master | CPT code → fee mapping | Hospital Admin |
| ICD → Package Master | ICD code → TPA package per TPA | TPA Coordinator |
| TPA Tariff Master | Per-TPA negotiated rates, sub-limits, exclusions | TPA Coordinator |
| Medicine Master | Drug → price + TPA formulary flag | Pharmacy |
| Consumable / Item Master | SKU → price + procedure bundle map | Stores |
| Procedure → Consumable Bundle | Implicit consumable inference (e.g., foley → catheter + tray + urobag) | Clinical Ops |
| TPA Policy Sub-limit Master | Room limit %, consult cap, drug sub-limit per policy | TPA Coordinator |

## 10. Hard Edge Cases to Handle

| Edge Case | Handling |
|---|---|
| **Implicit consumables** | Procedure → consumable bundle table (e.g., "put in foley" → 3 line items) |
| **Quantity ambiguity** | Use clinical defaults (e.g., "IV fluids" = 2L = 4×500ml bags) unless nursing notes override |
| **High-value implants** | Hard gate: never auto-bill; require invoice + model number before billing |
| **Multi-visit OP** | Aggregate consultation fees per doctor per visit; flag duplicates |
| **ER → IP handoff** | Bill split at admission timestamp; no cross-billing |
| **Multi-consultant IP** | Primary = full visit rate; secondary = referral/opinion rate; apply TPA cap if applicable |
| **OT time slabs** | Duration from OT start/end timestamps, not transcription |
| **Package vs itemized (TPA)** | Itemized for internal costing always; patient bill uses package rate |

> **Integration strategy:** Unizy pushes extracted line items as a structured payload to the hospital's existing HIS billing module via API. Unizy is the **extraction + mapping layer**, not a replacement for the HIS.

## 12. Phased Build Plan

### Phase 1 — OP Cash Billing
- **Scope:** Consultation + Lab + Pharmacy extraction for cash OP patients
- **Why first:** Highest volume, simplest structure, fastest to validate with hospital partners
- **Deliverables:**
  - Doctor identity → consultation fee lookup
  - Investigation orders → lab line items
  - Prescription → pharmacy line items
  - Itemized OP bill JSON payload

### Phase 2 — IP Cash Billing
- **Scope:** Room × LOS + ICD-based procedure + consumable bundle inference
- **Deliverables:**
  - Room allocation table integration
  - ICD code → procedure fee mapping
  - Consumable bundle inference engine
  - Itemized IP bill JSON payload

### Phase 3 — TPA / Insurance Layer
- **Scope:** ICD → TPA package mapping + pre-auth draft generation + NHCX-ready claim payload
- **Deliverables:**
  - TPA tariff master + ICD package master
  - Pre-auth auto-draft from transcription output
  - Dual-bill engine (itemized + TPA projection)
  - Confidence scoring + flagging logic

### Phase 4 — ER Billing
- **Scope:** Triage-based billing + emergency procedure extraction + ER→IP handoff logic
- **Deliverables:**
  - Triage level → fee tier mapping
  - ER procedure extraction
  - Bill split at admission timestamp
